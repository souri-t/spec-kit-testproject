## User Scenarios & Testing

### Primary User Story
- 管理者は設定画面でプロンプトと実行時刻を登録する。
- システムは毎日指定時刻にAIへ問い合わせ、結果を保存しWeb画面に表示する。

### Acceptance Scenarios
1. **Given** 設定画面でプロンプト・時刻を登録済み、**When** 指定時刻になる、**Then** AI問い合わせ結果がWeb画面に表示される
2. **Given** 設定変更後、**When** 次回実行時刻到来、**Then** 新しい設定でAI問い合わせ・表示される

### Edge Cases
設定が未登録の場合は、エラー表示（「設定未登録」など）を行う
AI問い合わせ失敗時は、一定回数自動リトライ後にエラー表示を行う
結果ファイル保存失敗時は、エラー表示のみ行う

AI問い合わせ失敗時は、一定回数自動リトライ後にエラー表示を行う

## Requirements

**FR-001**: システムは毎日指定時刻にAIへプロンプトを送信し、結果を取得する

**FR-003**: 設定はJSON形式で保存される
**FR-004**: AI問い合わせ結果はJSONファイルとして保存される
**FR-005**: 最新のAI結果をWeb画面で表示する
**FR-006**: 使用AIはOpen Routerとし、必要なAPIキー等の設定項目を設ける
**FR-007**: 設定未登録時はエラー表示（「設定未登録」など）を行う
